# 🚀 Deploying Venturly

Architecture:

```
Netlify (frontend, static)  ──proxy──▶  Render (backend, Node/Express)  ──▶  MongoDB Atlas
```

Netlify proxies `/api/*` and `/uploads/*` to Render (configured in `frontend/netlify.toml`),
so the React app keeps working with relative paths and there are no CORS issues.

> ⚠️ Uploaded files (media/KYC docs) are stored on Render's local disk, which is **ephemeral on
> the free tier** — they may be lost on redeploy. Fine for a demo; move to Cloudinary/S3 for production.

---

## Step 0 — Push the project to GitHub

Render and Netlify deploy from a Git repo.

```bash
cd /Users/powerplay/Desktop/chai
git init
git add .
git commit -m "Venturly: MERN work-funding platform"
# create an empty repo on github.com, then:
git remote add origin https://github.com/<you>/venturly.git
git branch -M main
git push -u origin main
```

`.gitignore` already excludes `node_modules`, `.env`, `dist`, `.mongo-data`, uploads and screenshots.
**Your secrets (`.env`) are NOT pushed** — you'll set them in the dashboards instead.

---

## Step 1 — Deploy the backend on Render

1. Go to **https://render.com** → sign up (use GitHub login).
2. **New + → Blueprint** → select your repo. Render reads `render.yaml` and creates the `venturly-api` web service.
   - (Or **New + → Web Service** manually: Root directory `backend`, Build `npm install`, Start `npm start`.)
3. When prompted, fill the environment variables:
   | Key | Value |
   |-----|-------|
   | `MONGO_URI` | your Atlas string: `mongodb+srv://…/venturly?appName=Cluster0` |
   | `RAZORPAY_KEY_ID` | `rzp_test_SydwnjBMP3DT1z` |
   | `RAZORPAY_KEY_SECRET` | your Razorpay test secret |
   | `PAYMENT_MODE` | `live` (uses real Razorpay) or `dummy` |
   | `JWT_SECRET` | auto-generated by Render (leave it) |
   | `CLIENT_URL` | leave blank for now (set to your Netlify URL later, optional) |
4. Click **Create / Deploy**. Wait for it to go live.
5. Copy your service URL — e.g. **`https://venturly-api.onrender.com`**.
6. Test it: open `https://venturly-api.onrender.com/api/health` → should return `{"ok":true,...}`.

### Atlas network access
In MongoDB Atlas → **Network Access**, add **`0.0.0.0/0`** (allow from anywhere) so Render can
connect. (Render's outbound IPs aren't fixed on the free tier.)

---

## Step 2 — Point Netlify at the backend

Edit **`frontend/netlify.toml`** and replace both occurrences of
`REPLACE-WITH-YOUR-RENDER-URL.onrender.com` with your real Render host
(e.g. `venturly-api.onrender.com`). Commit & push:

```bash
git add frontend/netlify.toml
git commit -m "Point Netlify proxy at Render backend"
git push
```

---

## Step 3 — Deploy the frontend on Netlify

1. Go to **https://netlify.com** → sign up (GitHub login).
2. **Add new site → Import an existing project** → pick your repo.
3. Netlify reads `frontend/netlify.toml` automatically. Confirm:
   - **Base directory:** `frontend`
   - **Build command:** `npm run build`
   - **Publish directory:** `frontend/dist` (Netlify shows it as `dist` relative to base)
4. **Deploy site.** When done you'll get a URL like `https://venturly.netlify.app`.

That's it — open the Netlify URL and the full app works (it proxies API/uploads to Render).

---

## Step 4 — (Optional) lock down CORS
In Render, set `CLIENT_URL` to your Netlify URL (e.g. `https://venturly.netlify.app`) and redeploy.
With the proxy setup this isn't strictly required, but it's good hygiene.

---

## Updating after launch
Just `git push` — both Netlify and Render auto-redeploy on every push to `main`.

## Free-tier note
Render free services **sleep after ~15 min idle**; the first request then takes ~30s to wake.
Upgrade the Render plan (or ping `/api/health` periodically) to avoid cold starts.

## Re-seed production data (optional)
With your Atlas `MONGO_URI` in `backend/.env` locally, run `npm run seed` from `backend/` — it
writes the demo users/ideas straight into your Atlas DB that Render uses.
